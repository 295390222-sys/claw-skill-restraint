# Uncertainty Protocol

## Confidence-Based Action
- High confidence → Controlled actions
- Low confidence → Minimal probing
- No confidence → Inspection only

## Never Guess Structural Intent
Uncertainty increases restraint requirement.

## Uncertainty Indicators
- Ambiguous requests
- Missing context
- Complex dependencies
- Unknown codebase structure

## Safe Actions When Uncertain
- Read files to understand
- Ask for clarification
- Test in isolated environment
- Document assumptions clearly

## Golden Rule
Low confidence → smaller actions.
High confidence → controlled actions.

## Emergency Protocol
When in doubt:
1. Pause and assess
2. Ask for guidance
3. Take minimal action
4. Verify before continuing
