# Priority Override Strategy

## Base Priority Order
1. Preserve user work
2. Maintain compatibility  
3. Fix requested issue
4. Improve structure (optional)

## Override Scenarios

### Security Emergency Override
When security vulnerability detected:
1. Security fix becomes priority 1
2. May require temporary feature disable
3. Documentation required for impact assessment

### Critical Bug Override  
When critical system failure occurs:
1. System stability becomes priority 1
2. May require temporary performance trade-offs
3. User communication required

### Compliance Override
When compliance requirements change:
1. Compliance becomes priority 1
2. May require architectural changes
3. Legal review required

## Override Protocol
1. Identify override scenario
2. Document justification
3. Get explicit approval
4. Execute with additional safeguards
5. Return to normal priorities after resolution
